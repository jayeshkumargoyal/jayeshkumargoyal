## ğŸ› ï¸ Setting Up Your Environment

### Prerequisites

- Git installed on your system
- A GitHub account
- Basic understanding of the project structure

### Fork and Clone

1. **Fork the repository** on GitHub
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/your-username/jayeshkumargoyal.git
   cd jayeshkumargoyal
   ```
3. **Add upstream remote**:
   ```bash
   git remote add upstream https://github.com/jayeshkumargoyal/jayeshkumargoyal.git
   ```

### Development Setup

#### For Web Development
```bash
# Install Node.js dependencies (if applicable)
npm install

# Install Python dependencies (if applicable)
pip install -r requirements.txt
```

#### For Python Projects
```bash
# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

## ğŸ¤ Contributing Guidelines

### How to Contribute

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes** and commit them
4. **Push to your fork**: `git push origin feature/your-feature-name`
5. **Create a Pull Request**

### Types of Contributions

- **Bug Reports**: Report issues and bugs
- **Feature Requests**: Suggest new features
- **Documentation**: Improve or add documentation
- **Code Contributions**: Submit code improvements
- **Design**: Contribute to UI/UX improvements

### Code Standards

- Write clean, readable code
- Follow existing code style
- Add comments for complex logic
- Use meaningful variable names
- Write tests for new features
